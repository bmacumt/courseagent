### 二、数据隔离方案

#### 2.1 隔离原则

```
知识库隔离规则：
- 教师A上传的文档 → 只有教师A能看到和删除
- 教师B上传的文档 → 只有教师B能看到和删除
- -默认：全局检索（展示知识融合能力）
- 可选：管理员设置按 owner_id 过滤（前端开关控制）
- 管理员：可查看所有文档（但不能删除）

作业隔离规则：
- 教师只能管理自己发起的作业
- 学生只能看到自己提交的作业
```

#### 2.2 表结构

```sql
users:
  id, username, password_hash, role, created_at

documents:
  id, filename, title, doc_type,
  owner_id,
  chunk_count,
  uploaded_at

assignments:
  id, teacher_id, title, description,
  grading_criteria(JSON),
  deadline, created_at

submissions:
  id, assignment_id, student_id,
  content, status,
  submitted_at

reports:
  id, submission_id,
  total_score,
  dimension_scores(JSON),
  feedback,
  references(JSON),
  created_at
```

**方案：统一 Collection + Metadata 过滤**

```
Collection: tunnel_knowledge

每个chunk的metadata:
{
  "doc_id": 123,

```

## 四、RAG核心架构（重点）

### 4.1 检索流程

```
用户问题
  ↓
向量检索（Chroma）
  ↓
BM25检索
  ↓
RRF融合
  ↓
Top-K截断（前端可调）
  ↓
BGE Reranker重排序
  ↓
LLM生成答案
```

------

### 4.2 多路检索融合（RRF）

采用 RRF（Reciprocal Rank Fusion）融合多路检索结果：

公式：

[
RRF(d) = \sum_{i=1}^{n} \frac{1}{k + rank_i(d)}
]

实现：

```python
def rrf_fusion(vector_docs, bm25_docs, k=60):
    scores = {}

    for rank, doc in enumerate(vector_docs):
        scores[doc.id] = scores.get(doc.id, 0) + 1 / (k + rank)

    for rank, doc in enumerate(bm25_docs):
        scores[doc.id] = scores.get(doc.id, 0) + 1 / (k + rank)

    return sorted(scores.items(), key=lambda x: x[1], reverse=True)
```

------

### 4.3 Top-K策略（前端可控）

前端参数：

```json
{
  "top_k": 15,
  "rerank_k": 5
}
```

推荐默认值：

```
vector_top_k = 20
bm25_top_k = 20
rrf_top_k = 15
rerank_top_k = 5
```

------

### 4.4 重排序（BGE Reranker）

流程：

```
RRF结果 → 构造(query, doc) → reranker打分 → 排序 → 取Top-N
```

作用：

- 提升语义匹配精度
- 修正embedding误差
- 优化长文本相关性

------

### 4.5 分块策略（优化版）

采用结构优先分块：

优先按：

- 标题层级
- 条款编号
- 段落结构

补充：

- 固定窗口（chunk_size=500, overlap=50）
