## 前后端联调最稳方案

------

### 一、联调前必须准备的材料

在给 AI 助手下指令之前，你需要先从后端拿到以下内容，**缺一不可**：

**1. 完整的 API 文档**（接口地址、请求方法、请求体、响应体）

```
例如：
POST /api/auth/login
Request:  { "username": string, "password": string }
Response: { "token": string, "user": { "id", "role", "real_name" } }
```

**2. 后端 BaseURL**（本地开发地址，如 `http://localhost:8000`）

**3. 认证方式**（JWT Token / Session Cookie / API Key，放在哪个 Header）

**4. 错误码规范**（如 `{ "code": 401, "message": "..." }`）

------

### 二、联调的正确顺序（非常重要）

```
第一步：封装 API 层（统一 axios/fetch 实例）
第二步：按模块逐个替换 Mock 数据
第三步：处理 Loading / Error 状态
第四步：联调测试，修复字段映射问题
```

**绝对不要**：同时替换所有 Mock，一次性改完，这会让 debug 变成噩梦。

------

### 三、给 AI 助手下指令的标准模板

#### 📌 第一条指令：建立 API 基础层

```
我有一个 React + TypeScript 项目，需要开始前后端联调。

后端 BaseURL：http://localhost:8000
认证方式：JWT，登录后将 token 存入 localStorage（key: "tunnel_auth_token"），
         后续请求在 Header 中携带 Authorization: Bearer <token>
错误响应格式：{ "detail": "错误信息" }（FastAPI 标准格式）

请帮我：
1. 安装 axios
2. 在 /src/app/api/client.ts 创建 axios 实例，配置：
   - baseURL
   - 请求拦截器（自动从 localStorage 读取 token 并注入 Authorization header）
   - 响应拦截器（统一处理 401 跳转登录页，其他错误抛出 message）
3. 在 /src/app/api/ 下按模块建立 API 文件，本次先只建 auth.ts，
   不要修改任何页面组件
```

------

#### 📌 第二条指令：替换登录接口（单个模块示例）

```
现在替换登录接口。

后端接口文档：
POST /api/auth/login
Request Body: { "username": string, "password": string }
Response 200: { "access_token": string, "token_type": "bearer", 
               "user": { "id": number, "username": string, 
                         "role": "admin"|"teacher"|"student", 
                         "real_name": string | null } }
Response 401: { "detail": "用户名或密码错误" }

请帮我：
1. 在 /src/app/api/auth.ts 实现 login() 函数
2. 修改 /src/app/context/AuthContext.tsx 中的 login 方法，
   调用真实接口，将 access_token 存入 localStorage
3. 保持其他所有页面的 Mock 数据不变，本次只动这两个文件
```

------

#### 📌 第三条指令：替换某个页面的数据（以作业列表为例）

```
现在替换教师作业列表页的数据，其他页面保持 Mock 不变。

后端接口：
GET /api/assignments?page=1&page_size=20
Response 200: {
  "items": [{
    "id": number,
    "title": string,
    "description": string,
    "deadline": string | null,
    "is_published": boolean,
    "submission_count": number,
    "created_at": string
  }],
  "total": number
}

当前前端文件：/src/app/pages/teacher/Assignments.tsx
当前使用 mockAssignments 数据

请帮我：
1. 在 /src/app/api/assignments.ts 新增 getAssignments() 函数
2. 修改 Assignments.tsx，用 useEffect + useState 替换 mock 数据，
   加上 loading 骨架屏状态和请求失败的 error 提示
3. 字段映射如有差异请告知，不要自行猜测
```

------

### 四、处理字段不一致的指令模板

这是联调最常见的问题，提前告诉 AI 怎么处理：

```
后端返回的字段名和前端 Mock 不一致：
- 后端 student_score → 前端用的是 total_score
- 后端 created_time（时间戳 number）→ 前端用的是 created_at（ISO string）

请在 /src/app/api/assignments.ts 的接口函数中做数据转换，
转换后的数据结构必须与现有 Mock 数据类型（mockData.ts 中的 interface）完全一致，
页面组件不需要改动
```

------

### 五、联调过程中的关键约束（每次都要加在指令里）

```
【约束条件，必须遵守】
1. 每次只改一个模块，不要动其他文件
2. 所有新增的 API 函数统一放在 /src/app/api/ 目录下
3. 接口函数必须有 TypeScript 类型定义（入参和返回值）
4. 出错时 console.error 打印完整错误，页面显示友好提示
5. 不要删除 Mock 数据文件，保留作为降级方案
6. 如果后端返回的数据结构你不确定，直接告诉我，不要猜
```

------

### 六、推荐的最终文件结构

```
/src/app/api/
  ├── client.ts          # axios 实例 + 拦截器（第一步建）
  ├── auth.ts            # 登录/登出
  ├── assignments.ts     # 作业相关
  ├── submissions.ts     # 提交相关
  ├── knowledge.ts       # 知识库相关
  └── admin.ts           # 管理员相关
```

------

### 七、一句话总结

> 

**每次只换一个接口，先建 API 层再改页面，字段不一致在 API 层转换，约束条件每次都附在指令里。** 这样即使出错，范围也被严格限定，回滚和 debug 都非常容易。
